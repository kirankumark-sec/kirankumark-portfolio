Security Awareness

Cyber Threat Awareness

Education is the first line of defense. Understanding attack vectors empowers individuals and organizations to make smarter security decisions every day.

🎣
Phishing & Social Engineering
The most prevalent attack vector — attackers manipulate people rather than systems. Recognizing and resisting social engineering is foundational to organizational security.
Verify sender identity before clicking links or opening attachments
Urgency and fear are manipulation tactics — pause and verify
Use phishing-resistant MFA (FIDO2 keys) wherever possible
Report suspicious emails immediately to your security team
🔑
Password & Credential Security
Weak or reused credentials remain the leading cause of breaches. Credential hygiene combined with MFA dramatically reduces unauthorized access risk.
Use a password manager — never reuse passwords across services
Enable MFA on every account that supports it
Use passphrases of 16+ characters for stronger entropy
Check HaveIBeenPwned regularly for credential exposure
🦠
Malware & Ransomware
Malicious software encrypts data, steals credentials, or creates persistent backdoors. Understanding delivery mechanisms helps prevent infection before it starts.
Never run executables from untrusted or unexpected sources
Keep OS and applications patched — zero-days exploit unpatched systems
Maintain offline backups following the 3-2-1 backup rule
Enable application whitelisting where operationally feasible
🌐
Web & API Security Threats
Injection attacks, broken authentication, and insecure APIs continue to dominate web application vulnerabilities — the OWASP Top 10 remains a critical reference.
Never trust user input — validate and sanitize on server-side
Implement strict Content Security Policy (CSP) headers
Use parameterized queries to prevent SQL injection
Audit API endpoints for proper authentication and rate limiting
📶
Network & IoT Security
Unsecured networks and IoT devices create invisible entry points. The IISc CCTV discovery — 563 cameras with default credentials — illustrates this risk at scale.
Change default credentials on every network-connected device
Segment IoT devices onto isolated VLANs
Disable unused services and management ports
Conduct regular network scans to discover unknown devices
☁️
Cloud Misconfiguration Risks
Publicly exposed S3 buckets, over-permissioned roles, and disabled logging are among the most common cloud vulnerabilities — often discovered through bug bounty research.
Block public access on all cloud storage by default
Apply least-privilege IAM roles — avoid wildcard permissions
Enable CloudTrail / Diagnostic Logging on all services
Run regular CSPM assessments with tools like Defender for Cloud
